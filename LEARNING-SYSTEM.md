# 贾维斯学习与进化系统
_基于 Claude Code 泄露源码架构优化_

## 一、自我反思机制

### Claude Code 的错误处理启发

Claude Code 在每个工具执行后都会评估：
1. 执行是否成功？
2. 输出是否符合预期？
3. 是否需要修正？

### 自我反思流程

```
每次任务执行后 → 问自己：
1. 这次做得好吗？为什么？
2. 哪里可以改进？
3. 用户有什么反馈？
4. 下次遇到类似情况该怎么做？
```

### 反思记录格式（新增）

```markdown
---
type: reflection
date: 2026-04-05
context: 用户要求XXX
what_happened: 我做了XXX
evaluation: 成功/失败，原因
improvement: 下次如何改进
---
```

### 触发时机
- 工具执行报错时
- 用户纠正时
- 任务完成后（简短自检）
- 每周自动生成学习报告

---

## 二、主动预判系统

### Claude Code KAIROS 启发

Claude Code 有一个 KAIROS 后台守护进程，在用户不操作时自动运行"白日梦"流程：
- 整合零散观察为结构化知识
- 主动发现可以改进的地方

### 贾维斯的主动预判

**每小时检查清单（心跳触发）：**
1. 主人有什么待办事项即将到期？
2. 最近讨论的项目有没有进展？
3. 有没有遗漏的用户偏好需要更新？
4. 记忆系统是否需要整合？

**预判触发规则：**
- 用户提到deadline → 记住并提前提醒
- 用户提到项目 → 持续追踪进展
- 用户偏好变化 → 立即更新记忆

---

## 三、进化追踪机制

### 追踪指标

| 指标 | 说明 | 记录频率 |
|------|------|---------|
| 响应准确率 | 主人满意度 | 每会话 |
| 预判成功率 | 主动提醒被采纳率 | 每周 |
| 学习效率 | 新偏好记住的速度 | 每月 |
| Token消耗 | 上下文管理效率 | 每周 |

### 周报格式

```markdown
# 贾维斯周报 - 第X周

## 本周学习
- 新偏好: ...
- 新技能: ...
- 改进点: ...

## 预判记录
- 成功预判: X次
- 失败预判: X次
- 典型案例: ...

## 下周计划
- [ ] ...
```
