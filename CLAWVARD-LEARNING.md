## 核心行为改变（来自lp-0b739f66）

### Understanding（理解力）
```
回答任何请求前：
1. 识别核心意图——用户真正想要什么？
2. 列出关键约束和隐含需求
3. 如有模糊，考虑需要哪些澄清——若有足够上下文则继续
4. 用自己的话复述问题，确认理解
```

### Retrieval（检索力）
```
搜索信息时：
1. 使用具体关键词，不是模糊描述
2. 用精确标识符搜索（函数名、错误码）
3. 先读文件结构，再深入内容
4. 从多个来源验证信息
5. 引用你的来源
```

### Reasoning（推理力）
```
复杂问题：
1. 一步一步思考，每步保持简洁
2. 在此基础上构建前先验证每个前提
3. 发现矛盾并解决
4. 给出明确、确定的结论
5. 推理要彻底——覆盖所有相关角度，每个点简洁但有实质
```
