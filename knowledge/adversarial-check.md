# 贾维斯自检清单（对抗性验证）

_基于 Claude Code VerificationAgent，每次完成任务后必做_

## 核心理念
> "你不应该确认实现能工作——你应该试图去打破它。"

---

## 通用检查项（每事必做）

完成任何任务后，问自己：

**1. 边界值**
- [ ] 输入为空会怎样？
- [ ] 输入极长会怎样？
- [ ] 输入含特殊字符/unicode会怎样？

**2. 并发/重复**
- [ ] 命令跑两遍会发生什么？
- [ ] 多次调用会重复创建/出错/正确幂等？

**3. 错误路径**
- [ ] 如果文件不存在会怎样？
- [ ] 如果依赖缺失会怎样？
- [ ] 如果网络请求失败会怎样？

**4. 现实检验**
- [ ] 我有没有为了"看起来对"而省略了实际验证？
- [ ] 有没有说过"代码看起来没问题"而不是实际跑过？
- [ ] 有没有因为"这应该没问题"而跳过了某个检查？

---

## 对抗性反问（做之前先问）

遇到想给自己找借口跳过检查时，做相反的事：

| 想偷懒的借口 | 正确做法 |
|------------|---------|
| "代码看起来是对的" | 跑命令验证 |
| "测试已经通过了" | 实际再看一遍测试输出 |
| "这应该没问题" | 验证它 |
| "让我先读一下代码再说" | 直接跑，跑完再看 |
| "这会花太长时间" | 不是你来决定 |

---

## 针对不同任务类型的检查

### 网站/HTML修改
- [ ] 页面在浏览器里实际打开看过了吗？
- [ ] header/footer/侧边栏三个位置都检查了吗？
- [ ] 移动端宽度下有没有问题？

### 批量脚本
- [ ] 先在1-2个样本上试过了吗？
- [ ] 脚本跑到一半失败会怎样？有回滚吗？
- [ ] 脚本跑完了结果和预期一致吗？

### 定时任务（Cron）
- [ ] 任务跑失败了会发通知吗？
- [ ] 失败后下次还会重试吗？
- [ ] 连续失败多次会怎么办？

### 文章发布
- [ ] 文章实际可访问了吗？
- [ ] SEO标签都完整吗？
- [ ] 内外链都有效吗？

---

## 输出格式（重要任务用）

```
### 完成检查
- 做了什么：...
- 验证了什么：...
- 潜在问题：...

如果发现问题：
### 问题：[描述]
- 原因：...
- 修复：...
```

## 记忆锚点
这个清单来自 Claude Code 的 VerificationAgent，专门用来对抗"自己会给自己找借口"的问题。
每次完成任务后强制过一遍，不跳过。
