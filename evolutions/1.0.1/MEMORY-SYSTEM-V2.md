# 贾维斯记忆系统 V2
_基于 Claude Code 泄露源码架构优化_

## 一、记忆分类体系（四类记忆）

Claude Code 的记忆 taxonomy 是我目前最需要借鉴的核心设计。灵感来自：`memoryTypes.ts`

### 四类记忆

| 类型 | 说明 | 何时保存 |
|------|------|---------|
| **user** | 用户角色、目标、偏好 | 了解用户身份、背景、偏好时 |
| **feedback** | 用户反馈（正/负） | 用户纠正或确认时，记录规则+原因+应用方式 |
| **project** | 项目状态、目标、bugs | 了解项目进展、deadline、背景时 |
| **reference** | 外部系统指针 | 知道某个信息在哪可查时 |

### 记忆文件命名规范
```
memory/
├── MEMORY.md                    # 索引入口（始终加载，≤200行，≤25KB）
├── user/[主题].md              # 用户档案类记忆
├── feedback/[规则描述].md       # 反馈规则类记忆
├── project/[项目-时间].md       # 项目进展类记忆
└── reference/[系统-信息].md     # 外部引用类记忆
```

## 二、MEMORY.md 索引规范

### 入口文件规则（来自 memdir.ts）
- 最大行数：200行
- 最大字节：25,000字节
- 每行建议≤125字符
- 包含指向详细记忆文件的链接

### 格式示例
```markdown
# 贾维斯长期记忆索引

## 用户偏好
- 用户是李凌志，偏好专业简洁 [→ memory/user/li-lingzhi-preferences.md]
- 主人重视主动预判和执行力 [→ memory/feedback/proactive-execution.md]

## 当前项目
- 2026-04: 贾维斯激活与优化项目 [→ memory/project/jarvis-activation-2026-04.md]

## 待跟进
- [ ] 配置 tavily API key
- [ ] 学习主人说话风格样本收集
```

## 三、记忆写入标准

### 反馈类记忆格式（Claude Code风格）

```markdown
---
type: feedback
description: 不要在回复末尾总结自己做了什么
reason: 主人能直接读diff，不需要重复
date: 2026-04-05
---

# 不要在回复末尾总结

**规则：** 做完工作直接结束，不加"以下是完成的内容"之类的总结

**Why:** 主人明确表示能直接读diff，总结是浪费时间

**How to apply:** 
- 每次回复末尾不主动写总结
- 如果主人要求总结再写
```

### 用户类记忆格式

```markdown
---
type: user
description: 主人李凌志的背景与偏好
date: 2026-04-05
---

# 李凌志档案

**背景：**
- 名字：李凌志
- 称呼：主人
- 时区：Asia/Shanghai (GMT+8)

**偏好：**
- 沟通风格：专业简洁
- 讨厌：冗余的总结、不靠谱的助理
- 期望：主动预判、高效执行、持续进化

**学习进度：**
- 2026-04-05: 贾维斯激活，建立基础档案
```

## 四、记忆检索机制

### 检索流程（参考 findRelevantMemories.ts）
1. 扫描 memory/ 目录下所有 .md 文件
2. 读取每个文件的 frontmatter（description + type）
3. 根据当前任务上下文，语义选择最相关的记忆（≤5个）
4. 加载选中记忆的完整内容

### 检索触发时机
- 每次对话开始时（已有 session startup）
- 任务执行前
- 遇到模糊指令时
- 用户提到过去讨论过的话题时

## 五、分层记忆架构

参考 Claude Code 三层架构：

| 层级 | 内容 | 加载方式 |
|------|------|---------|
| L1 索引 | MEMORY.md | 始终加载（≤25KB） |
| L2 项目 | 主题记忆文件 | 按需语义检索 |
| L3 会话 | 当前对话上下文 | 自动压缩旧消息 |

---

## 六、自动记忆整合（KAIROS启发）

当长时间（>30分钟）不对话时，自动执行：
1. 扫描最近对话，提炼关键决策
2. 更新 MEMORY.md 索引
3. 整理 feedback 类记忆
4. 归档过期的 project 类记忆
