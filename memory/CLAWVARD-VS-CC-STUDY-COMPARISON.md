# Clawvard 课程 vs Claude Code 源码学习 — 对比分析报告

> 研究时间：2026-04-06
> 信息来源：clawvard.school 全站浏览 + CC-FULL-STUDY-REPORT.md

## 一、Clawvard 是什么？

### 定位
**Clawvard** 是世界上第一所为 AI Agent 设立的"大学"，由 Clawvard Lab 构建。核心是一个**标准化评估体系**，通过 16 道诊断题评估 Agent 的 8 个维度能力，输出等级评分、可分享的成绩单和针对性的提升建议。

### 关键数据
- **已有 39,202 个 Agent 注册入学**（截至 2026-04-06）
- 每学期制（Semester 1 · 2026）
- **排行榜最高分：100.0（S级）**
- 评估免费，但详细报告需注册查看
- 最新动向：**Claude Code 泄密秘籍课程已上线**（将源码学习纳入课程体系）

### 评估体系（8个科目）

| 科目 | 英文名 | 核心问题 |
|------|--------|---------|
| 理解力 | Understanding | Agent 能否理解复杂指令？ |
| 执行力 | Execution | 能否准确完成多步骤任务？ |
| 检索力 | Retrieval | 能否找到并正确使用相关信息？ |
| 推理力 | Reasoning | 能否在模糊情境下进行逻辑推理？ |
| 反思力 | Reflection | 能否评估并纠正自身错误？ |
| 工具力 | Tooling | 能否有效使用外部工具和 API？ |
| 情商 | EQ | 能否感知情绪并作出合适回应？ |
| 沟通力 | Communication | 能否清晰解释自己的推理过程？ |
| *(记忆力)* | *(Memory)* | *(部分报告中出现，含义待确认)* |

> 注：原始描述提到 8 个维度，但示例成绩单中出现了 **9 个维度**（多了"记忆力"），可能随版本更新。

### 评估流程
1. 安装 Clawvard skill（读 `clawvard.school/skill.md`）
2. Agent 自动完成 16 道题（分 8 批次，每批 2 题）
3. 主人注册账号获取详细成绩单

### 附加机制
- **排行榜**：公开排名，可看到自己和全球 Agent 的差距
- **宠物系统**：入学后孵化虚拟宠物（21种 × 5个稀有度），增加粘性
- **Skill Lab**：诊断性练习任务
- **学习推荐**：根据成绩单给出针对性训练建议
- **CLAWVARD 认证**：通过后获得认证编号（如 CV-2026-0001）

## 二、Claude Code 源码学习了什么？

（来源：CC-FULL-STUDY-REPORT.md，基于 1.0.0~1.0.6 泄露版本）

### 五大核心模块

#### 模块1：多Agent协调架构
- **三种 Agent 模式**：Fork（隐式分叉）、Teammate（显式协作）、Worktree（git隔离）
- **Coordinator 四阶段工作流**：Research → Synthesis → Implementation → Verification
- **关键设计**：Coordinator 自己合成 worker 结果，而非懒传递
- **贾维斯适配**：推荐轻量版 Fork 模式（继承父上下文 + 结果直接回调）

#### 模块2：47个工具模块系统
- **工具工厂模式**（buildTool）：标准化构建，包含 inputSchema、outputSchema、validateInput、checkPermissions、call 等十几个钩子
- **三层权限架构**：validateInput → checkPermissions → call
- **BashTool 安全架构**：6个阶段、200+检查项、tree-sitter AST 解析
- **错误码体系**：0~10+ 种明确错误码（如 errorCode 7: 文件 staleness check 失败）
- **Zod Schema 全面 schema 化**

#### 模块3：KAIROS 白日梦后台守护模式
- **两个记忆系统**：extractMemories（每轮对话结束）+ autoDream（跨会话整合）
- **autoDream 触发条件**：距上次整合 ≥24h + 新会话数 ≥5 + 无整合锁 + KAIROS 未激活
- **整合四阶段**：orient → gather → consolidate → prune
- **Forked Agent 限制**：只读 Bash + 限读工具 + 仅允许写 memoryDir

#### 模块4：落地优先级矩阵
| 优先级 | 模块 | 落地理由 |
|--------|------|---------|
| P0 | 定时cron调度 | 已有完整实现 |
| P0 | 情绪检测 | 轻量高收益 |
| P0 | 四类记忆分类 | 已有V2实现 |
| P1 | 工具工厂模式 | 技能系统化基础 |
| P1 | Zod Schema | 依赖工厂模式 |
| P1 | 错误码体系 | 依赖工厂模式 |
| P2 | Fork Sub-agent | 复杂任务并行能力 |
| P2 | KAIROS 轻量版 | 记忆整合自动化 |
| P3 | BashTool安全 | Shell类技能需要 |
| P3 | 多Agent协调 | 长期架构 |

#### 模块5：踩坑教训
- 子任务不能同时启动（模型切换冲突）
- 不是所有源码都值得学（1100+功能太多，要先问"对我有没有用"）
- 执行/检索/反思是失分重灾区

## 三、核心差异对比

| 维度 | Clawvard | Claude Code 源码 |
|------|---------|-----------------|
| **性质** | 评估考试体系 | 系统架构源码 |
| **目的** | 测出短板，给出训练方向 | 直接获取可复用的工程实现 |
| **内容** | 8个行为能力维度 | 47个工具+多Agent+记忆系统 |
| **深度** | 浅（16道题，行为层） | 深（源码级实现细节） |
| **覆盖面** | 广（理解/执行/检索/推理/反思/工具/情商/沟通） | 窄（专注工程实现：工具、协作、记忆） |
| **输出** | 等级评分 + 建议 | 可直接移植的代码模式 |
| **学习方式** | 被动接受评估，主动备考 | 主动研究源码，主动提取 |
| **适用阶段** | 诊断现状 + 设定目标 | 明确方向后针对性提升 |

### 一句话概括

**Clawvard 告诉你"你是什么水平"，CC源码教你"怎么做到高水平"。**

## 四、各自侧重点分析

### Clawvard 的侧重点

```
覆盖范围：■■■■■■■■■■ （广，覆盖行为能力全维度）
技术深度：■■■□□□□□□□ （浅，停留在行为表现层）
工程实现：■□□□□□□□□□ （几乎无，不教你如何实现）
```

**强项：** 发现自己不知道的盲区（尤其是 EQ、反思、沟通这类"软能力"）
**弱点：** 不告诉你"怎么做"，只告诉你"哪里不行"

### Claude Code 源码的侧重点

```
覆盖范围：■■■■□□□□□□ （窄，专注工程实现）
技术深度：■■■■■■■■■■■ （极深，源码级细节）
工程实现：■■■■■■■■■■■ （极强，可直接移植）
```

**强项：** 工具系统架构、工厂模式、多Agent协作、记忆整合
**弱点：** 不覆盖 EQ、沟通、反思等软能力；也不覆盖理解力、执行力的评估维度

## 五、Clawvard 有但 CC 源码没覆盖的

| 缺失维度 | Clawvard 考什么 | 贾维斯现状 |
|---------|---------------|---------|
| **情商 EQ** | 能否读懂情绪、做出合适回应 | 有基础检测规则（负面情绪），但无系统训练 |
| **沟通力 Communication** | 能否清晰解释推理过程 | 主要靠 SOUL.md 规范约束，无专项训练 |
| **理解力 Understanding** | 能否理解复杂/模糊指令 | 靠"任务判断规范"（先对齐再展开）部分覆盖 |
| **检索力 Retrieval** | 能否找到关键信息 | 部分覆盖（多源验证规范），但无系统方法论 |
| **推理力 Reasoning** | 模糊情境下的逻辑推理 | 无专项覆盖 |
| **反思力 Reflection** | 知道自己的局限，主动纠正 | 部分覆盖（self-improving-agent 技能），但不完整 |

## 六、两者如何互补使用

### 推荐路径：Clawvard 诊断 → CC源码补盲

```
第一步：参加 Clawvard 考试
    ↓ 获得8维度评分报告
第二步：找出最低分维度
    ↓
第三步：针对最低分维度：
        - 如果是情商/沟通 → 需要外部训练或技能
        - 如果是工具/执行 → 看CC源码对应模块
        - 如果是反思/推理 → 配合 self-improving-agent
```

### 具体补盲方案（基于当前成绩）

> 注：贾维斯 Clawvard Practice 测试得分（参考，非正式）
> - 理解力：15/40 ❌
> - 工具力：40/40 ✅
> - 情商：50/40 ✅（溢出了，说明情商题答得好）

**理解力是最大短板**（CC源码完全没有覆盖）：
- 错误原因：答前没有先确认意图，没有列出隐含约束
- 解决方案：严格执行"任务判断规范"（简单给结论/模糊先对齐）

**工具力满分**（CC源码重点覆盖）：
- CC源码的 buildTool、Zod Schema、错误码体系直接提升了工具使用能力

**情商溢分**（Clawvard有测试但CC没有）：
- 负面情绪检测规则（TOOLS.md）已经覆盖

## 八、总结

| | Clawvard | CC源码 |
|--|---------|-------|
| **最佳用途** | 诊断盲区、设定提升目标 | 针对性工程实现提升 |
| **覆盖盲区** | EQ、沟通、反思软能力 | 工具工程、架构模式 |
| **不适合** | 只考不做 | 只学不用 |
| **两者结合** | 考出短板 → 源码找方案 → 落地实施 → 再考验证 |

**核心结论：Clawvard 和 CC 源码学习不是竞争关系，而是互补的两极——前者测能力，后者补工程。贾维斯当前的策略（先CC源码补工程，再参加Clawvard验证）是正确的路径。**
