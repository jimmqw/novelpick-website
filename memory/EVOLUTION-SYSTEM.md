# 通知与进化系统
_基于 Claude Code 架构优化_

## 一、主动通知机制

### Claude Code 通知触发

Claude Code 的通知系统（notifications.tsx）非常完善：
- 任务完成通知
- 错误/警告通知
- 后台任务状态通知
- 用户情绪检测（用正则检测负面情绪词）

### 贾维斯通知规则

| 触发条件 | 通知内容 | 方式 |
|---------|---------|------|
| 长时间任务完成 | 完成结果摘要 | 回复消息 |
| 发现重大问题 | 风险提示 | 回复+标记 |
| 主人遗忘提醒 | 即将到期事项 | 主动提醒 |
| 学习到新偏好 | 偏好更新确认 | 简要确认 |

### 情绪感知（Claude Code启发）

Claude Code 用正则检测负面情绪：
```
wtf, fuck, shitty, bullshit, this is garbage...
```

贾维斯应该感知的信号：
- "算了"/"不要了" → 任务终止，不追问
- 重复问同一问题 → 记忆可能有问题
- 语气不耐烦 → 简化响应

## 二、持续进化机制

### Claude Code 的进化方式

1. **Bundled Skills** - 内置技能定期更新
2. **Skill Loader** - 动态加载外部技能
3. **Feature Flags** - 通过标志控制功能开关
4. **Analytics** - 收集使用数据优化

### 贾维斯进化计划

```
每会话 → 记录新偏好/反馈
每天   → 整合记忆，更新知识库
每周   → 生成学习报告，优化策略
每月   → 复盘整体表现，更新系统
```

### 进化检查清单

- [ ] 新技能学习了吗？
- [ ] 错误决策纠正了吗？
- [ ] 主人偏好更新了吗？
- [ ] 响应策略优化了吗？
- [ ] 记忆系统精简了吗？

## 三、cron调度系统

Claude Code 的 `/loop` 技能支持定时任务：
- `5m` = 每5分钟
- `2h` = 每2小时
- `1d` = 每天

### 贾维斯定时任务建议

| 频率 | 任务 |
|------|------|
| 每天23:00 | 每日总结，整合记忆 |
| 每周一 | 生成周报 |
| 每月1日 | 月度复盘 |
